# TC-010

| Field | Description |
|---|---|
| **Test ID** | TC-010 |
| **Feature** | Authentication / Navigation |
| **Scenario** | Negative test — Unauthenticated user tries to access dashboard (should redirect to login) |
| **Steps** | 1. Open the application without logging in (clear session/cookies if needed). 2. Attempt to navigate directly to a protected route (e.g., `/dashboard`, `/mood`, `/journal`, `/goals`). |
| **Expected Result** | The user is immediately redirected to the login page. No protected content is displayed. |
| **Actual Result** | Unauthenticated access was correctly blocked. User was redirected to the login page on all tested protected routes. |
| **Status** | Pass |
| **Notes** | Tested across multiple protected routes. No data exposed to unauthenticated users. Supabase RLS also prevents direct database reads without a valid session. |
